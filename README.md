Микросервис для расписания врачей. Умеет создавать разовые и периодические задачи.

## Что под капотом (Периодичность)
Считаем даты через `rrule`. В настройках можно задать:
* Дату старта
* Тип повторения
* Сколько раз повторять
* До какой даты повторять

**Как решена проблема 31-го числа:** если выбрали 29, 30 или 31 число, а в месяце дней меньше (например, в феврале), задача автоматом ставится на последний день месяца.

## Эндпоинты
**Списки и серии:**
* `POST /api/v1/tasks/` — создать разовую задачу или серию (передав `recurrence`).
* `GET /api/v1/tasks/` — посмотреть расписание (с фильтрами по датам, чтобы не класть базу).
* `DELETE /api/v1/tasks/templates/{template_id}` — удалить шаблон и все его невыполненные задачи разом.

**Одиночные задачи:**
* `GET /api/v1/tasks/{task_id}` — получить одну задачу.
* `PUT /api/v1/tasks/{task_id}` — обновить конкретную задачу (изменить текст или статус).
* `DELETE /api/v1/tasks/{task_id}` — удалить только одну задачу (разовую или одну из серии).

## Подключение к БД
Подставьте нужные значения в DB_USER, DB_PASSWORD, DB_IP, DB_PORT в variables
```bash
DATABASE_URL = postgresql+psycopg://DB_USER:DB_PASSWORD@DB_IP:DB_PORT/DB_NAME
```
